1. Key Notes

  1. Set parameters. In my case, I set \(t_{max} = 1000\), \(k_b = 0.2\), \(k_d = 0.2\), \(k_1 = 1\), \(k_b = 0.01\), \(k_3 = 1\), \(k_4 = 0.02\).

  2. Set \(\delta t\) to 0.001 (or comparably small number, much smaller than \(\lambda\)). Create a vector that spaced the time from 0 to tmax with this interval. This is to “resample” each trace and align them against time.

  3. After aligning against time, it’s now possible to calculate mean.

2. Algorithm

  1. initialize matrix to save all traces.

  2. for each trace, if time is smaller than \(t_{max}\):

    • calculate \(w_1\) to \(w_6\)
    • decide waiting time from exponential distribution, where \(\lambda = 1/sum(w)\).
    • update everything during this waiting time. This step is critical because only one event should happen, thus the number for other molecules should not change.
    • generate another random number, decide which kind of event should happen.
    • update the status based on the event decided by previous step.

Results for Q3

In here, I used function code to test parameters.

  • Parameter used for bursting: \(t_{max} = 1000\), \(k_b = 0.01\), \(k_u = 1\), \(k_1 = 5\), \(k_b = 0.05\), \(k_3 = 1\), \(k_4 = 0.01\).
  • Parameter used for bursting: \(t_{max} = 1000\), \(k_b = 0.1\), \(k_u = 0.1\), \(k_1 = 1\), \(k_b = 0.01\), \(k_3 = 1\), \(k_4 = 0.01\).

  • Therefore, whether the bursting effect is highly depends on the ratio of \(k_b\) and \(k_u\). if \(k_u\) is much larger than \(k_b\), and the absolute value of these two parameters is comparable to others, then every time the gene is activated it will be turned off pretty soon.

  • Another important parameter is \(k_1\). This parameter should be large enough that during the short period of turning on gene, there are RNAs being produced. In this case the \(k_1\) should be roughly larger than \(k_u\). RNA degradation term, \(k_2\), should also be significantly smaller than \(k_1\) to allow RNA to accumulate
